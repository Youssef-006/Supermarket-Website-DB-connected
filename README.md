A front-end web application for managing and visualizing supermarket database operations. The project simulates a database management interface, allowing users to interact with core entities through a structured and responsive UI.

Built with HTML, CSS, JavaScript, and TailwindCSS, the system supports basic CRUD-like functionality (adding and deleting records) across multiple categories such as customers, products, orders, suppliers, cashiers, deliveries, and discounts.

Features
Management of multiple entities (customers, products, orders, etc.)
Dynamic forms that adapt based on the selected data type
Add and delete operations with immediate UI updates
Tab-based navigation for switching between datasets
Live statistics showing record counts for each category
Automatic calculations (e.g., cashier salary based on hourly pay)
Responsive design with a clean, modern layout
Database Integration

The interface is designed to work alongside an Oracle database, acting as a front-end layer for interacting with structured data.

Tech Stack
HTML5
CSS3 (TailwindCSS + custom styling)
Vanilla JavaScript (state-driven rendering)
Oracle Database (backend integration)
Purpose

This project was developed as part of a database systems course to demonstrate how a front-end application can represent and manage relational data in a user-friendly way.
